# راهنمای سریع رفع مشکل مقالات

## مشکل اصلی
فایل `.env` وجود ندارد، بنابراین Firebase پیکربندی نشده و مقالات از Firestore لود نمی‌شوند.

## راه‌حل سریع

### مرحله 1: ایجاد فایل .env
```bash
cp .env.example .env
```

### مرحله 2: پر کردن اطلاعات Firebase
1. به [Firebase Console](https://console.firebase.google.com/) بروید
2. پروژه خود را انتخاب کنید (یا یک پروژه جدید ایجاد کنید)
3. به **Project Settings** (⚙️) بروید
4. در بخش **Your apps**، اپلیکیشن Android را انتخاب کنید
5. مقادیر را از config object کپی کنید
6. در فایل `.env` قرار دهید

### مرحله 3: Import کردن مقالات
```bash
node scripts/seedArticlesWeb.js
```

این اسکریپت:
- 7 نویسنده را اضافه می‌کند
- 14 مقاله (7 دری + 7 پشتو) را اضافه می‌کند

### مرحله 4: Restart کردن Expo
```bash
# در ترمینال Ctrl+C بزنید
npx expo start --clear
```

### مرحله 5: بررسی در اپلیکیشن
1. اپلیکیشن را Refresh کنید
2. به تب "مقالات" بروید
3. مقالات باید نمایش داده شوند

## بررسی لاگ‌ها

برای دیباگ، در ترمینال Metro bundler به دنبال این لاگ‌ها بگردید:
- `[DEBUG]` - لاگ‌های دیباگ
- `[Articles]` - لاگ‌های مربوط به مقالات
- `[Firebase]` - لاگ‌های Firebase

اگر خطای "Index missing" دیدید، نگران نباشید - کد به صورت خودکار از fallback query استفاده می‌کند.
